# PRD-00: STS2 Community Stats Mod — 总体产品需求文档

> 版本：1.0 | 日期：2026-04-07  
> 目标：完成服务器部署、修复战斗统计缺陷、打磨 UI，使 Mod 达到可公开发布的质量

---

## 1. 产品概述

STS2 Community Stats 是一个杀戮尖塔 2 的数据统计 Mod。它收集玩家的局内数据（卡牌选取、事件选择、商店购买、战斗贡献等），上传到后端服务器进行聚合分析，然后在游戏内向所有安装 Mod 的玩家展示社区统计数据（选取率、胜率、危险度等）。

### 1.1 项目目标

- **对内**：借助社区数据做数值设计 / 数据驱动的分析和学习
- **对外**：为 STS2 社区提供有价值的统计工具
- **当前阶段目标**：不新增功能，把现有功能做完做好，达到可公开发布的状态

### 1.2 "可公开发布"的定义

1. 服务器稳定运行，能接收上传和响应查询
2. 战斗统计数据在所有已知场景下准确无遗漏
3. UI 信息展示清晰、不遮挡游戏画面、不引发视觉 Bug

## 2. 术语表

### 2.1 Mod 架构术语

| 术语 | 英文 | 定义 |
|------|------|------|
| **Harmony 补丁** | Harmony Patch | 通过运行时方法拦截修改游戏行为，不修改游戏文件 |
| **Prefix 补丁** | Prefix | 在原方法执行前运行的拦截代码 |
| **Postfix 补丁** | Postfix | 在原方法执行后运行的拦截代码 |
| **ModInitializer** | ModInitializer | STS2 官方 Mod 加载器的入口标记属性 |
| **反编译代码** | Decompiled Code | 从 sts2.dll 用 ILSpy 反编译的游戏源码，仅供参考，不编译 |

### 2.2 数据收集术语

| 术语 | 英文 | 定义 |
|------|------|------|
| **局（Run）** | Run | 从开始到胜利/失败的一次完整游戏，是数据上传的最小单位 |
| **战斗贡献（Contribution）** | Contribution | 一张卡/遗物/药水在战斗中造成的伤害、格挡、抽牌等效果的量化归因 |
| **直接伤害** | Direct Damage | 卡牌效果直接造成的伤害 |
| **归因伤害** | Attributed Damage | 由力量（Power）间接造成的伤害（如中毒），归因到施加该力量的来源 |
| **修正伤害** | Modifier Damage | 由数值修正器贡献的伤害增量（如力量加成、易伤翻倍） |
| **有效格挡** | Effective Block | 实际吸收了伤害的格挡量（FIFO 消耗池追踪） |
| **削弱减伤** | Mitigated by Debuff | 因敌人身上的虚弱（Weak）效果而减少的受伤量 |
| **增益减伤** | Mitigated by Buff | 因玩家的 Buffer/无实体（Intangible）等效果而减少的受伤量 |
| **力量削减减伤** | Mitigated by Str Reduction | 因降低敌人力量而减少的受伤量 |
| **升级增量** | Upgrade Delta | 卡牌升级带来的额外伤害/格挡，归因到升级来源（篝火/事件） |
| **贡献来源** | Contribution Source | 产生贡献的实体：卡牌（Card）、遗物（Relic）、药水（Potion） |
| **贡献上下文** | Contribution Context | ContributionMap 维护的当前活跃来源栈，用于归因间接效果 |
| **Osty** | Osty | STS2 特有的召唤物机制，有独立 HP 池，需要 LIFO 栈追踪归因 |
| **来源子条** | Sub-bar | 贡献图表中的缩进子条目，表示由某个来源间接产生的效果（如无限刀刃产生的碎片） |

### 2.3 统计展示术语

| 术语 | 英文 | 定义 |
|------|------|------|
| **选取率** | Pick Rate | 卡牌/遗物在被提供时被玩家选择的比例 |
| **胜率** | Win Rate | 选择该卡牌/遗物的局中最终胜利的比例 |
| **危险度** | Danger | 某个遭遇的平均受伤量和死亡率 |
| **贡献图表** | Contribution Chart | 水平条形图，展示每个来源在各类别（伤害/防御/抽牌/能量/星星/治疗）中的贡献 |
| **批量统计包** | Bulk Stats Bundle | 服务器返回的预计算统计数据集合，包含某角色在某版本某升级范围下的所有统计 |
| **筛选面板** | Filter Panel | F9 打开的设置面板，可按升级等级、版本、胜率筛选统计数据 |

### 2.4 后端术语

| 术语 | 英文 | 定义 |
|------|------|------|
| **预计算** | Precompute | 后台定时任务，每 10 分钟聚合一次全量统计数据并缓存到 Redis |
| **升级范围** | Ascension Range | 预计算的升级等级分组：0-4 / 5-9 / 10-14 / 15-19 / 20 |
| **ID 迁移** | ID Migration | 跨游戏版本的卡牌/遗物 ID 重命名映射表 |
| **离线队列** | Offline Queue | 客户端在服务器不可达时缓存上传数据，待恢复后重传 |

## 3. 系统架构总览

### 3.1 数据流

```
玩家游戏中                   Mod 客户端                      服务器
─────────                   ──────────                      ──────
打出卡牌 ──→ CombatHistoryPatch 拦截 ──→ CombatTracker 累计
获得遗物 ──→ RelicHookPatcher 拦截   ──→ ContributionMap 归因
使用药水 ──→ PotionUsedPatch 拦截    ──→ ContributionAccum 累加
                                          │
战斗结束 ──→ CombatLifecyclePatch ──→ RunContributionAggregator 汇总
                                          │
局结束 ──→ OnMetricsUpload ──→ RunDataCollector 打包 ──→ ApiClient.Upload()
                                                              │
                                                         POST /v1/runs
                                                              │
                                                    ┌─────────▼─────────┐
                                                    │   FastAPI Server   │
                                                    │  ┌──────────────┐ │
                                                    │  │  PostgreSQL   │ │
                                                    │  └──────────────┘ │
                                                    │  ┌──────────────┐ │
                                                    │  │    Redis      │ │
                                                    │  └──────────────┘ │
                                                    └───────────────────┘
                                                              │
查看奖励/商店/事件 ──→ Mod 请求 GET /v1/stats/* ──→ 返回统计数据
                          │
                     StatsCache (内存+磁盘) 缓存响应
                          │
                     各 Patch 渲染 StatsLabel / Chart
```

## 4. 交互方式

### 4.1 数据上传（对玩家透明）

1. 玩家正常进行一局游戏，Mod 在后台静默收集数据
2. 一局结束（胜利或失败）时，游戏触发 `OnMetricsUpload` 事件
3. Mod 打包本局所有数据，通过 HTTPS POST 上传到服务器
4. 如果服务器不可达，数据存入离线队列，下次启动时重传
5. 玩家无需任何操作，上传行为对游戏无影响

### 4.2 卡牌选取率/胜率查看

1. 战斗胜利后进入卡牌奖励界面
2. 每张可选卡牌下方显示 **选取率** 和 **胜率** 标签
3. 数据来源：StatsCache → 服务器 `/v1/stats/bulk` → 对应角色和升级范围
4. 首次加载时可能有短暂延迟，后续使用缓存

### 4.3 遗物胜率查看

1. 鼠标悬停在遗物上方
2. 遗物 Tooltip 中额外显示 **社区胜率** 和 **选取率**

### 4.4 事件选项统计查看

1. 进入事件房间，显示选项按钮
2. 每个选项旁显示 **社区选择率** 和 **选择后胜率**

### 4.5 战斗贡献图表查看

1. 战斗结束后，自动弹出贡献面板（可配置是否自动弹出）
2. 面板显示两个 Tab：**本场战斗** 和 **本局汇总**
3. 每个 Tab 包含多个类别的水平条形图：
   - 伤害（Damage）：蓝色=卡牌直接、浅蓝=归因、紫色=修正加成
   - 防御（Defense）：绿色=格挡、浅绿=削弱减伤、橙色=力量削减
   - 抽牌（Draw）、能量（Energy）、星星（Stars）、治疗（Healing）
4. 条形按贡献值从大到小排列
5. 有子来源的条目可展开显示子条

### 4.6 地图危险度标注

1. 地图界面中，已知遭遇的节点旁显示 **危险度标签**
2. 显示内容：平均受伤量 和 死亡率百分比

### 4.7 商店/升级/移除统计

1. 商店界面：卡牌旁显示社区购买率
2. 篝火升级选择：卡牌旁显示社区升级率
3. 移除卡牌选择：卡牌旁显示社区移除率

### 4.8 筛选面板（F9）

1. 按 F9 打开筛选面板
2. 可设置：升级等级范围、游戏版本、仅看高胜率玩家数据
3. 修改筛选条件后，所有统计标签自动刷新

### 4.9 全局统计面板（F8）

1. 按 F8 打开全局统计面板
2. 显示当前筛选条件下的聚合数据概览

## 6. 总验收标准

### AC-01: 端到端数据流
- [ ] 服务器在云端稳定运行
- [ ] 客户端完成一局后数据成功上传
- [ ] 其他玩家安装 Mod 后能查询到统计数据
- [ ] 服务器不可达时客户端不崩溃，数据进入离线队列

### AC-02: 战斗统计准确性
- [ ] 所有 6 个角色（铁甲/静默/机器人/观者/漫游者/死灵）的基础卡牌贡献正确统计
- [ ] 遗物被动贡献（伤害/格挡/抽牌/能量）正确归因
- [ ] 药水使用贡献正确归因
- [ ] 力量间接伤害（中毒、荆棘等）正确归因到来源
- [ ] Osty 召唤物的伤害/防御正确归因
- [ ] 末杀伤害不遗漏
- [ ] 自伤卡牌（放血、祭祀等）正确显示为负防御
- [ ] 数值修正加成（力量/灵巧/易伤/虚弱）正确分拆显示

### AC-03: UI 质量
- [ ] 贡献图表不遮挡关键游戏 UI
- [ ] 统计标签在各分辨率下不溢出
- [ ] 长卡牌名/遗物名不截断或溢出
- [ ] 颜色编码在深色和浅色背景下均可辨识

### AC-04: 稳定性
- [ ] Mod 不导致游戏崩溃
- [ ] Mod 不导致游戏卡顿（网络请求异步，不阻塞主线程）
- [ ] 卸载 Mod 后游戏正常运行
