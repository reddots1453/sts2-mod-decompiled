# PRD-00: STS2 Community Stats Mod — 总体产品需求文档

> 版本：1.0 | 日期：2026-04-07  
> 目标：完成服务器部署、修复战斗统计缺陷、打磨 UI，使 Mod 达到可公开发布的质量

---

## 1. 产品概述

STS2 Community Stats 是一个杀戮尖塔 2 的数据统计 Mod。它收集玩家的局内数据（卡牌选取、事件选择、商店购买、战斗贡献等），上传到后端服务器进行聚合分析，然后在游戏内向所有安装 Mod 的玩家展示社区统计数据（选取率、胜率、危险度等）。

### 1.1 项目目标

- **对内**：借助社区数据做数值设计 / 数据驱动的分析和学习
- **对外**：为 STS2 社区提供有价值的统计工具
- **当前阶段目标**：不新增功能，把现有功能做完做好，达到可公开发布的状态

### 1.2 "可公开发布"的定义

1. 服务器稳定运行，能接收上传和响应查询
2. 战斗统计数据在所有已知场景下准确无遗漏
3. UI 信息展示清晰、不遮挡游戏画面、不引发视觉 Bug

| 术语 | 英文 | 定义 |
|------|------|------|

| **Prefix 补丁** | Prefix | 在原方法执行前运行的拦截代码 |

### 2.2 数据收集术语

| 术语 | 英文 | 定义 |
|------|------|------|
| **局（Run）** | Run | 从开始到胜利/失败的一次完整游戏，是数据上传的最小单位 |
| **战斗贡献（Contribution）** | Contribution | 一张卡/遗物/药水在战斗中造成的伤害、格挡、抽牌等效果的量化归因 |
| **直接伤害** | Direct Damage | 卡牌效果直接造成的伤害 |
| **归因伤害** | Attributed Damage | 由力量（Power）、中毒（poison）或一些能力牌、遗物等造成的间接伤害，归因到施加力量来源或伤害根本来源 |
| **修正伤害** | Modifier Damage | 由数值修正器贡献的伤害增量（如力量加成、易伤翻倍） |
| **有效格挡** | Effective Block | 实际吸收了伤害的格挡量（FIFO 消耗池追踪） |
| **削弱减伤** | Mitigated by Debuff | 因敌人身上的虚弱（Weak）效果而减少的受伤量 |
| **增益减伤** | Mitigated by Buff | 因玩家的 Buffer/无实体（Intangible）等效果而减少的受伤量 |
| **力量削减减伤** | Mitigated by Str Reduction | 因降低敌人力量而减少的受伤量 |
| **升级增量** | Upgrade Delta | 在战斗内卡牌升级带来的额外伤害/格挡，归因到升级来源（某些卡牌、药水、遗物） |
| **贡献来源** | Contribution Source | 产生贡献的实体：卡牌（Card）、遗物（Relic）、药水（Potion） |
| **贡献上下文** | Contribution Context | ContributionMap 维护的当前活跃来源栈，用于归因间接效果 |
| **奥斯提Osty** | Osty | STS2 角色亡灵契约师特有的召唤物机制，有独立 HP 池，需要 LIFO 栈追踪归因 |
| **来源子条** | Sub-bar | 贡献图表中的缩进子条目，表示由某个来源（主条目）生成/变化的卡牌的贡献效果，显示在主条目下（如无限刀刃产生的小刀） |

### 2.3 统计展示术语

| 术语 | 英文 | 定义 |
|------|------|------|
| **选取率** | Pick Rate | 卡牌在被提供时被玩家选择的比例 |
| **胜率** | Win Rate | 卡组中具有该卡牌/遗物的局中最终胜利的比例 |
| **危险度** | Danger | 某个遭遇的平均受伤量和死亡率 |
| **贡献图表** | Contribution Chart | 水平条形图，展示每个来源在各类别（伤害/防御/抽牌/能量/辉星/治疗）中的贡献 |
| **批量统计包** | Bulk Stats Bundle | 服务器返回的预计算统计数据集合，包含某角色在某版本和用户筛选条件下（进阶范围和胜率）的所有统计 |

| 术语 | 英文 | 定义 |
|------|------|------|

## 4. 交互方式

### 4.1 数据上传（对玩家透明）

1. 玩家正常进行一局游戏，Mod 在后台静默收集数据
2. 一局结束（胜利或失败）时，游戏触发 `OnMetricsUpload` 事件
3. Mod 打包本局所有数据，通过 HTTPS POST 上传到服务器
4. 如果服务器不可达，数据存入离线队列，下次启动时重传
5. 玩家无需任何操作，上传行为对游戏无影响

### 4.2 卡牌选取率/胜率查看

1. 战斗胜利后进入卡牌奖励界面，或者事件/遗物的卡牌奖励界面
2. 每张可选卡牌右下方显示 **选取率** 和 **胜率** 标签
3. 数据来源：StatsCache → 服务器 `/v1/stats/bulk` → 对应角色和筛选范围
4. 首次加载时可能有短暂延迟，后续使用缓存

### 4.3 遗物胜率查看

1. 鼠标悬停在遗物上方
2. 遗物 Tooltip 中额外显示 **胜率** （没有**选取率**因为遗物的获得基本都是被动的，无法主动选择）

### 4.4 事件选项统计查看

1. 进入事件房间，显示选项按钮
2. 每个选项旁显示 **社区选择率** 和 **选择后胜率**

### 4.5 战斗贡献图表查看

1. 战斗结束后，自动弹出贡献面板（可配置是否自动弹出）；按 F8 也打开和关闭
2. 面板显示两个 Tab：**本场战斗** 和 **本局汇总**
3. 每个 Tab 包含多个类别的水平条形图：
   - 伤害（Damage）：蓝色=卡牌直接、浅蓝=间接归因、紫色=修正加成
   - 防御（Defense）：绿色=格挡、浅绿=削弱减伤、橙色=力量削减
   - 抽牌（Draw）、能量（Energy）、星星（Stars）、治疗（Healing）
4. 条形按贡献值从大到小排列，最多列出10个贡献源，其他贡献源归入“其他”显示
5. 有子来源的条目展开显示子条sub-bar
6. 贡献面板右上角标出？帮助按钮，鼠标放上去可以看到贡献图表的功能具体说明（图表水平条形图颜色的具体含义，计算了哪些间接伤害/减伤的来源，贡献计算的先后顺序等）

### 4.5.1 战斗贡献计算需求

#### 总体原则

战斗贡献系统的核心目标是：**战斗中发生的每一点伤害、格挡、治疗等效果，都能追溯到造成该效果的根本来源（某张卡牌、某个遗物、某瓶药水）。** 玩家在贡献面板中看到的数字之和，应与战斗中实际发生的总量一致，不多不少。

#### 伤害归因需求

**直接伤害**：打出一张攻击卡造成的伤害，归属于该卡牌。

**数值修正拆分**：伤害中由力量、易伤、双倍伤害、易伤修正等修正器贡献的增量部分，需要从总伤害中拆分出来，分别归属到施加该修正的来源。修正器分为加法增量和乘法增量，拆分后，总伤害 =（基础伤害 + 各加法修正器增量a1,a2,a3,...求和）* 各乘法修正器系数b1,b2,b3,..乘积 。例如：打出一张攻击牌，力量药水带来的力量加成部分（加法）归属药水，痛击bash带来的易伤增伤效果（乘法，系数0.5）归于痛击，残酷增加对易伤状态敌人伤害的增伤效果（乘法，但和易伤增伤处于同一区间，系数0.25或0.5，取决于是否升级）归于残酷这张牌，攻击牌本身的基础伤害归属卡牌。
对于加法修正器，贡献为增量a1；对于独立乘区的乘法修正器，贡献为 总伤害 - 总伤害/（1+乘法修正系数b1）；对于非独立乘区的乘法修正器（如残酷，纸蛙等这样直接改变易伤基础系数的实体），贡献为 ( 总伤害 - 总伤害/（1+乘法区间总修正系数b0 + b1 +...+ bn）) * 当前乘法修正器b1/(乘法区间总修正系数b0 + b1 +...+ bn)。
其中易伤等增伤状态具有层数和不同来源，正确归因逻辑是：先施加增伤状态的实体，先被计算贡献。某些修正器的增量也会受到其他修正的作用，如力量的伤害增加会进一步受到易伤和双倍伤害的

**升级增量**：战斗内通过临时升级卡牌获得的额外伤害/格挡，需要独立拆分并归属到执行升级的来源（如武装、升级药水或某些遗物）。

**间接伤害**：非卡牌直接造成的伤害（如中毒、荆棘反伤、充能球、灾厄Doom 等），需要追溯到施加该效果的根本来源。例如：中毒伤害归属到施毒的那张卡，荆棘伤害归属到提供荆棘效果的遗物或力量。

**末杀伤害**：敌人被最后一击杀死时，伤害值应为敌人实际剩余 HP，而非卡牌面板伤害值。最后一击不得被遗漏或重复计算。

**自伤**：玩家对自己造成的伤害（如放血、祭祀类卡牌）需要单独记录为负防御值，归属于打出该卡的来源，不计入正向伤害统计。

**来源归因优先级**：当间接效果发生时，系统需要按上下文判断根本来源。优先级从高到低为：明确指定的卡牌来源 → 当前正在打出的卡牌 → 当前使用的药水 → 当前执行钩子的遗物 → 当前触发的力量的施加来源 → 当前触发的充能球的引导来源。如果所有上下文都没有，则标记为"未追踪"。

#### 防御归因需求

**格挡归因**：多个来源依次提供的格挡，在被伤害消耗时，按照先获得先消耗的顺序（FIFO）归属。例如：先打出"防御A"获得 5 格挡，再打出"防御B"获得 5 格挡，敌人攻击 7 点——则 5 归属"防御A"，2 归属"防御B"。只统计实际吸收了伤害的有效格挡量，未使用的格挡不计入。

**格挡修正拆分**：灵巧、聚焦等修正器带来的额外格挡量需要从总格挡中拆分，归属到提供修正的来源。

**虚弱减伤**：敌人身上的虚弱效果导致攻击力降低，减少的伤害量归属到施加虚弱的来源。

**增益减伤**：玩家身上的 Buffer（完全抵消一次攻击）和无实体（将伤害降为 1）所避免的伤害量，归属到施加该增益的来源。

**力量削减减伤**：通过降低敌人力量而避免的伤害量，归属到执行力量削减的来源。如果多个来源都削减了同一敌人的力量，按各自削减量的比例分配。

#### Osty（亡灵契约师召唤物）归因需求

Osty 是亡灵契约师角色的特殊召唤物，有独立的 HP 池，需要专门的归因逻辑：

**Osty 承伤防御**：Osty 替玩家承受伤害时，消耗 Osty 的 HP 归属到恢复该 HP 的来源。恢复顺序按后恢复先消耗（LIFO）计算——最近一次恢复的 HP 最先被伤害消耗。

**Osty 攻击伤害**：Osty 对敌人造成的伤害归属到触发该次攻击的卡牌，同时标记为"来自 Osty"以便在图表中作为该卡牌的子条目显示。

**Osty 死亡**：Osty 被击杀时，其剩余未消耗的 HP 从击杀来源中扣除（负防御）。

#### 其他贡献类别需求

**抽牌**：额外抽牌（不含每回合正常抽牌）归属到触发抽牌的来源。

**能量**：额外获得的能量归属到提供能量的来源。

**星星（漫游者特有）**：星星贡献归属到触发星星获得的来源。

**治疗**：战斗中的治疗归属到触发治疗的来源；战斗外的治疗（遗物被动回血、休息点、事件等）也需追踪归属，计入全局汇总。

#### 子来源（Sub-bar）需求

当卡牌通过游戏机制发生变形或生成新卡时（如无限刀刃生成碎片、原始力量变为巨石），生成卡牌的贡献应作为原始来源的子条目显示，而非独立条目。系统需要追踪卡牌之间的生成/变形关系。

#### 数据生命周期需求

**单场战斗**：每场战斗独立追踪贡献数据，战斗结束后保留快照供"本场战斗"面板显示。

**全局汇总**：每场战斗的数据累加到全局汇总中，供"本局汇总"面板显示。战斗外的治疗等非战斗贡献也计入全局汇总。

**上传精简**：上传到服务器时，仅包含聚合后的精简字段（如合并的总伤害、总格挡），不上传本地展示用的细分字段（如修正伤害、修正格挡、升级增量、自伤、父来源标记等）。服务端不需要区分修正器的细分来源。

#### 容错需求

**无法归因时**：如果某个伤害/格挡效果无法追溯到具体来源，应记入"未追踪"类别，不丢弃数据，保证总量一致。

**特殊卡牌交互**：追索之刃（SeekingEdge）的非主目标伤害、聚焦（Focus）对充能球的加成等特殊机制需要专门处理，不能简单套用通用逻辑。

### 4.6 地图危险度标注

1. 地图界面中，已知遭遇的节点旁显示 **危险度标签**
2. 显示内容：平均受伤量 和 死亡率百分比

### 4.7 商店/升级/移除统计

1. 商店界面：卡牌旁显示社区购买率
2. 篝火升级选择：卡牌旁显示社区升级率
3. 移除卡牌选择：卡牌旁显示社区移除率

### 4.8 筛选面板（F9）

1. 按 F9 打开筛选面板
2. 可设置：根据进阶等级范围、游戏版本、胜率来筛选获取玩家数据
3. 修改筛选条件后，所有统计标签自动刷新
4. 可修改语言选项中文/English

## 5. 当前需完成的三大工作

### 5.1 服务器部署（→ PRD-01）

服务器代码已完成，需要实际部署到云服务器并验证端到端数据流。

### 5.2 战斗统计修复与测试（→ PRD-02）

经人工测试发现多处数据遗漏和不准确，需要系统性测试并修复。

### 5.3 UI 打磨（→ PRD-03）

## 6. 总验收标准

### AC-01: 端到端数据流
- [ ] 服务器在云端稳定运行
- [ ] 客户端完成一局后数据成功上传
- [ ] 其他玩家安装 Mod 后能查询到统计数据
- [ ] 服务器不可达时客户端不崩溃，数据进入离线队列

### AC-02: 战斗统计准确性
- [ ] 所有 5 个角色（铁甲/静默/机器人/漫游者/亡灵契约师）的卡牌贡献正确统计
- [ ] 遗物被动贡献（伤害/格挡/抽牌/能量）正确归因
- [ ] 药水使用贡献正确归因
- [ ] 间接伤害正确归因到来源
- [ ] Osty 召唤物的伤害/防御正确归因
- [ ] 战斗结束前最后一次伤害的贡献不遗漏
- [ ] 自伤卡牌（放血、祭祀等）正确显示为负防御，但不计入防御百分比计算
- [ ] 数值修正加成（力量/灵巧/易伤/虚弱）正确分拆显示
- [ ] 变化、生成卡牌sub-bar正确显示
- [ ] 不同类型的条状图颜色正确显示

### AC-03: UI 质量
- [ ] 贡献图表不遮挡关键游戏 UI，不遮挡卡牌奖励界面
- [ ] 统计标签在各分辨率下不溢出
- [ ] 长卡牌名/遗物名不截断或溢出
- [ ] 颜色编码在深色和浅色背景下均可辨识

### AC-04: 稳定性
- [ ] Mod 不导致游戏崩溃
- [ ] Mod 不导致游戏卡顿（网络请求异步，不阻塞主线程）
- [ ] Mod 能正常加载不报错
- [ ] 卸载 Mod 后游戏正常运行
