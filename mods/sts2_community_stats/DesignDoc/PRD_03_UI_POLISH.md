# PRD-03: UI 打磨与改进

> 版本：1.0 | 日期：2026-04-07  
> 依赖：ContributionPanel、ContributionChart、StatsLabel、FilterPanel（均已实现）

---

## 1. 概述

现有 UI 功能可用但体验不佳。本 PRD 聚焦于贡献面板、统计标签、筛选面板的视觉和交互打磨，不新增功能。

---

## 2. 术语表

| 术语 | 定义 |
|------|------|
| **贡献面板（ContributionPanel）** | 战斗结束后显示的主面板，包含贡献图表，有"本场战斗"和"本局汇总"两个 Tab |
| **贡献图表（ContributionChart）** | 水平条形图组件，按类别展示每个来源的贡献值 |
| **统计标签（StatsLabel）** | 附着在卡牌/遗物/事件选项旁的小标签，显示选取率/胜率等数据 |
| **筛选面板（FilterPanel）** | F9 打开的设置面板，控制统计数据的筛选条件 |
| **条形（Bar）** | 图表中的一个水平矩形，宽度代表贡献值 |
| **子条（Sub-bar）** | 缩进的条形，表示间接来源（如无限刀刃产生的碎片） |
| **颜色编码** | 不同来源类型用不同颜色区分：蓝=卡牌、金=遗物、青=药水等 |
| **Tooltip** | 鼠标悬停在 UI 元素上时弹出的详情浮窗 |

---

## 3. 交互方式

### 3.1 贡献面板

#### 当前行为
- 战斗结束后自动弹出
- 显示水平条形图，按贡献值排序
- 有"本场战斗"和"本局汇总"两个 Tab

#### 需要改进的交互

**3.1.1 面板定位和大小**
- 面板应居中显示，占屏幕约 60% 宽度、70% 高度
- 面板不可拖拽（避免遮挡问题）
- 面板外区域点击可关闭面板
- 面板右上角有关闭按钮（×）

**3.1.2 Tab 切换**
- "本场战斗"和"本局汇总"Tab 位于面板顶部
- 当前选中 Tab 有高亮下划线
- 切换 Tab 时内容即时更新，无加载延迟

**3.1.3 类别折叠**
- 每个类别（伤害/防御/抽牌/能量/星星/治疗）有标题栏
- 标题栏显示类别名称 + 该类别的总值
- 点击标题栏可折叠/展开该类别的条形列表
- 默认全部展开

**3.1.4 条形交互**
- 鼠标悬停在条形上时，显示 Tooltip：
  ```
  [来源名称]（[来源类型]）
  直接伤害：XXX
  归因伤害：XXX
  修正加成：XXX
  总计：XXX
  ```
- 条形末尾显示数值文本（不仅靠宽度表示）
- 值为 0 的来源不显示（避免空条形）

**3.1.5 子条展开**
- 有子来源的条形右侧显示展开图标（▶）
- 点击展开图标显示子条（缩进 + 较浅颜色）
- 再次点击折叠子条

**3.1.6 滚动**
- 如果条形数量超出面板高度，面板内容可垂直滚动
- 滚动条在右侧，样式与游戏 UI 一致

### 3.2 统计标签

#### 当前行为
- 附着在卡牌/遗物/事件选项旁显示选取率/胜率

#### 需要改进的交互

**3.2.1 标签布局**
- 标签不与卡牌名称或费用重叠
- 标签字体大小适中（不过小也不过大）
- 标签背景半透明深色，确保在各种卡牌图案上可读

**3.2.2 数据加载状态**
- 数据加载中时显示"加载中..."或旋转图标
- 数据加载失败时显示"暂无数据"
- 不显示裸露的 "0%" 或 "NaN"

**3.2.3 标签内容格式**
```
选取率: 45.2%  胜率: 52.1%  (n=1,234)
```
- 百分比保留一位小数
- 样本数用 `n=` 前缀，千位分隔符
- 样本数过少时（n < 50）标签显示为浅色/斜体，提示数据可能不具参考性

**3.2.4 各界面标签位置**
| 界面 | 标签位置 |
|------|---------|
| 卡牌奖励 | 卡牌下方居中 |
| 商店卡牌 | 价格标签下方 |
| 遗物悬停 | Tooltip 底部新增一行 |
| 事件选项 | 选项按钮右侧 |
| 篝火升级 | 卡牌下方居中 |
| 篝火移除/商店移除 | 卡牌下方居中 |

### 3.3 筛选面板（F9）

#### 需要改进的交互

**3.3.1 面板布局**
- 面板从屏幕右侧滑入
- 宽度约 300px，高度自适应
- 半透明暗色背景

**3.3.2 筛选项**
```
┌─────────────────────────┐
│  ⚙ 统计筛选              │
│                          │
│  升级等级                 │
│  [0] ──●────── [20]      │
│  当前：5 - 15            │
│                          │
│  游戏版本                 │
│  [▼ 最新版本 ]            │
│                          │
│  ☐ 仅显示高胜率玩家数据    │
│                          │
│  [ 应用 ]  [ 重置 ]       │
└─────────────────────────┘
```

**3.3.3 反馈**
- 修改筛选条件并点击"应用"后，所有统计标签立即刷新
- 刷新期间统计标签显示加载状态
- "重置"恢复默认筛选条件

## 4. 验收标准

### AC-01: 贡献面板
- [ ] 面板居中显示，大小合适，不遮挡游戏关键 UI
- [ ] Tab 切换正常，内容即时更新
- [ ] 条形按值排序，末尾显示数值
- [ ] 鼠标悬停显示详情 Tooltip
- [ ] 值为 0 的来源不显示
- [ ] 子条可展开/折叠
- [ ] 内容超出时可滚动
- [ ] 点击面板外或关闭按钮可关闭面板
- [ ] 类别标题栏可折叠/展开

### AC-02: 统计标签
- [ ] 各界面标签位置正确，不与游戏 UI 重叠
- [ ] 标签在不同分辨率（1080p / 1440p / 4K）下均可读
- [ ] 数据加载中/失败时有友好提示，不显示 NaN 或空白
- [ ] 百分比保留一位小数，样本数有千位分隔
- [ ] 小样本数据有视觉提示（浅色/斜体）

### AC-03: 筛选面板
- [ ] F9 打开/关闭面板
- [ ] 滑块正确控制升级范围
- [ ] 修改筛选后统计标签刷新
- [ ] "重置"恢复默认值

### AC-04: 颜色与可读性
- [ ] 条形颜色编码一致（蓝=卡牌、金=遗物、青=药水）
- [ ] 自伤为红色
- [ ] 所有文本在深色背景上可读（对比度足够）
- [ ] 子条颜色比父条更浅，层级可辨

### AC-05: 性能
- [ ] 贡献面板打开/关闭无卡顿
- [ ] 大量条形（50+）时滚动流畅
- [ ] 统计标签渲染不影响游戏帧率

---

## 5. 技术约束

- UI 基于 Godot 节点树（PanelContainer、VBoxContainer、HBoxContainer 等）
- 修改仅限 `src/UI/` 目录
- 颜色定义集中在 `ContributionChart.cs` 的常量中
- 标签附着逻辑在各 Patch 文件中（CardRewardScreenPatch、ShopPatch 等）
- 不使用游戏内部 UI 样式表（Mod 无法访问），用代码设置样式
- 分辨率适配通过 Godot 的锚点（Anchor）和容器自动布局实现
