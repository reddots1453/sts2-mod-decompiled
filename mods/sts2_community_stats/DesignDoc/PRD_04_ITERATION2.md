# PRD-04: Stats the Spire — 第二轮迭代需求文档

> 版本：3.0 | 日期：2026-04-09  
> UI 设计约束：参见 `UI_STYLE_GUIDE.md`  
> 前置：PRD-00（总体需求）、PRD-02（战斗统计，已完成）  
> 目标：扩展 mod 功能覆盖面，从"战斗贡献统计"扩展为"全面数据分析工具"

---

## 1. 迭代目标

第一轮迭代完成了战斗贡献统计核心功能（36/36 测试通过）。第二轮迭代目标：

1. **Mod 重命名**：重命名为 "Stats the Spire"，契合模组内容
2. **Compendium 集成**：在百科大全的卡牌图书馆和遗物收藏中展示社区统计
3. **概率透视**：问号房间遭遇概率、药水掉落概率、卡牌掉落概率、商店价格、敌人意图状态机
4. **实时贡献**：战斗中实时更新贡献图表（而非战斗结束后）
5. **个人数据分析**：在角色数据界面显示胜率趋势、死因分析、路径统计、先古遗物选择率
6. **Run History 增强**：在局回顾中展示本局详细统计和贡献图表回看
7. **功能开关**：每个子功能可独立启用/禁用
8. **多人兼容**：基础兼容性测试
9. **PRD-00 补全**：修正第一轮遗留的需求不明确项

## 2. 术语表（新增）

| 术语 | 英文 | 定义 |
|------|------|------|
| **胜率浮动** | Win Rate Delta | 持有该遗物的局胜率与当前筛选数据源的平均局胜率的偏差值 |
| **先古遗物** | Ancient Relic | 先古之民 Ancients 事件提供的遗物选择，类似 STS1 的 Boss 遗物，有极高分析价值 |
| **问号房间** | Unknown Room | 地图上的 ? 节点，随机决定遭遇类型（事件/怪物/宝箱/商店） |
| **怜悯机制** | Pity System | 问号房间遭遇概率调整机制，未抽中的遭遇类型概率逐次增加 |
| **意图状态机** | Intent State Machine | 怪物 AI 的行动决策系统，由状态节点和转移规则组成 |
| **实时贡献** | Real-time Contribution | 战斗中每次打牌/用药水后即时刷新贡献图表 |
| **生涯统计** | Career Stats | 聚合玩家所有历史局数据的统计指标 |
| **路径统计** | Path Stats | 按 Act 分层的游戏行为计数（抓卡数、商店数等） |

## 3. 功能需求

### 3.1 Mod 重命名

- 名称从 "STS2 Community Stats" 改为 **"Stats the Spire"**
- 涉及：manifest.json 的 `name` 字段、UI 标题、日志输出
- `id` 字段 "sts2_community_stats" 保持不变（机器标识，避免破坏已有安装）

### 3.2 Compendium 卡牌图书馆统计

1. 打开百科大全 → 卡牌图书馆
2. 在卡牌图书馆，每张已解锁卡牌点击打开大图后，**右侧方**显示统计标签（不覆盖卡牌）
3. 显示指标：**选取率** / **胜率** / **购买率** / **升级率**
4. 数据来源：StatsCache → 服务器批量统计（同卡牌奖励界面的数据源）
5. 滚动/切换筛选时标签跟随刷新
6. 未解锁卡牌不显示统计

**UI 设计**：
- 注入位置：卡牌大图打开时的回调（非 `InitGrid`），创建独立 `NModCardLibraryStats` 节点（与游戏内建 `NCardLibraryStats` 共存不冲突）
- 统计面板在卡牌大图**右侧**，VBoxContainer 4 行
- 字号 11px，标签 cream #FFF6E2，值 gold #EFC851
- 所有百分比显示到小数点后 1 位

```
┌──────────────┐ ┌────────────────┐
│              │ │ 选取率: 45.2%  │
│  [卡牌大图]  │ │ 胜率:   52.1%  │
│              │ │ 购买率: 12.3%  │
│              │ │ 升级率: 68.5%  │
└──────────────┘ └────────────────┘
```

### 3.3 Compendium 遗物收藏统计

1. 打开百科大全 → 遗物收藏
2. 鼠标悬停已解锁遗物时，显示 **胜率** 、 **胜率浮动** 和 **购买率**
3. 胜率浮动 = 该遗物持有局胜率 - 当前筛选数据源的平均局胜率
4. 正值绿色（如 "+3.2%"），负值红色（如 "-1.5%"）
5. 未解锁/未发现遗物不显示统计

**UI 设计**：
- 注入位置：`NRelicCollectionEntry.OnFocus()` postfix → 在现有 HoverTip 下方追加统计行
- 使用游戏 tooltip 系统（NHoverTipSet，360px 宽）
- 分隔线 + 胜率 + 胜率浮动 + 购买率
- 浮动值正 = green #7FFF00，负 = red #FF5555

```
┌─── 遗物 Tooltip (360px) ───┐
│ [遗物名称]                  │
│ [遗物描述]                  │
│ ─────────────────────────── │
│ 胜率: 52.3% (+2.3%)        │
│ 购买率: 15.1%              │
└─────────────────────────────┘
```

### 3.4 遗物胜率浮动（全局）

- 不仅在 Compendium 中显示，所有遗物 tooltip（背包悬停、商店、Boss 奖励等）均显示胜率浮动
- 格式：`胜率 52.3% (+2.3%)` — 括号内为浮动值
- 社区平均胜率暂用 mock 值（服务端部署后从 API 获取）

**UI 设计**：
- Hook: `IHoverTip.GetHoverTipContents()` postfix
- BBCode 内联着色：`"胜率 52.3% [color=#7FFF00](+2.3%)[/color]"`

### 3.5 事件选项简化

- 事件选项旁仅显示 **选择率**，移除选择后胜率
- 格式：`"选择率: 67.8%"` cream 色 11px

### 3.6 实时贡献更新

1. 战斗中按 F8 可随时打开/关闭贡献面板
2. 面板在打开状态下，**每次打出卡牌或使用药水后自动刷新**
3. 刷新内容：当前战斗的实时贡献数据（非快照）
4. 防抖：两次刷新间隔不小于 500ms
5. 战斗结束后行为不变（自动弹出最终结果）

### 3.7 每回合平均伤害

- 贡献面板标题区域显示：**"每回合平均伤害: X"**
- 计算：本场战斗总伤害 ÷ 已经过回合数
- 本局汇总 Tab 显示：所有战斗的总伤害 ÷ 总回合数

**UI 设计**：
- 注入位置：ContributionPanel 标题行下方，新增 HBoxContainer
- 标签 cream 12px，值 gold 12px

### 3.8 问号房间遭遇概率

1. 地图界面中，鼠标悬停在**未到达的 ? 节点**上时
2. 显示当前各遭遇类型的概率：
   - 事件 X% | 怪物 Y% | 宝箱 Z% | 商店 W%
3. 概率受怜悯机制影响，每次走过 ? 房间后概率重新分配
4. 数据来源：游戏内 `UnknownMapPointOdds` 的当前概率值

**UI 设计**（参考 STS1 InfoMod "Next [?] Floor" 截图）：
- 悬停未到达的 ? 节点时弹出 InfoMod 风格深色面板
- 标题 white 14px + 副标题 gray 11px + 分隔线 + 类别列表
- 各遭遇类别名使用不同颜色区分（事件 green、怪物 red、精英 orange、商店 gold、宝箱 aqua）
- 概率值 cream 12px，所有百分比显示到小数点后 1 位
- 面板样式：bg rgba(0.08,0.08,0.12,0.95)、border 1px rgba(0.3,0.4,0.6,0.4)、radius 6px

```
┌────────────────────────┐
│ 下一个 [?] 房间         │
│ 当前遭遇概率            │
│ ────────────────────── │
│ 事件      63.8%        │
│ 怪物      10.0%        │
│ 精英       0.0%        │
│ 商店       3.0%        │
│ 宝箱       2.0%        │
└────────────────────────┘
```

### 3.9 药水掉落概率

1. 显示位置：**战斗界面屏幕顶部边角**（类似 STS1 mod 截图），药水图标 + 当前概率百分比
2. 显示内容：
   - 当前概率（如 "40%"）
   - 精英战斗额外加成（如 "精英: 52.5%"）
   - 悬停展开：未来 1/2/3 场战斗至少掉落一次药水的累计概率
3. 数据来源：游戏内 `PotionRewardOdds.CurrentValue`
4. 概率受怜悯机制影响：掉落后 -10%，未掉落 +10%

**UI 设计**（参考 STS1 InfoMod "Potion Chance" 截图）：
- 常态：右上角固定 anchor，药水图标 + gold 色百分比（紧凑）
- 悬停展开：InfoMod 风格深色面板，列表显示未来 N 场累计概率
- 面板样式同 §3.8

```
常态：[药水图标] 40.0%

悬停展开：

│ Potion Chance      │
│ 多场累计概率        │

│ 2 场内    80.0%    │
│ 3 场内    94.0%    │
│ 4 场内    98.8%    │

└────────────────────┘
```

### 3.10 敌人意图状态机

1. 战斗中，鼠标悬停在敌人意图图标上时
2. 弹出面板显示该怪物的 AI 状态机：
   - **当前状态**：意图图标 + 伤害值
   - **箭头** → 指向下一步可能的状态
   - **简单转移（FollowUp）**：直接显示下一个动作图标
   - **随机分支**：列出每个分支的概率%、使用约束（≤N 次，表示最多连续使用 N 次）、动作图标（使用游戏意图系统原图标）+ 伤害值
   - **条件分支**：显示各分支的动作图标（条件为 lambda 无法显示为文本）
   - **循环箭头** ↩ 表示回到某个状态
3. 面板样式：深色半透明背景，与游戏 tooltip 风格一致

**UI 设计**（参考 STS1 意图状态机 Mod Jaw Worm 截图）：
- 注入位置：`NIntent.OnHovered()` postfix → 在敌人上方弹出状态机面板
- 意图图标使用游戏原生 NIntent 纹理，图标旁自动显示伤害/格挡数字，**不需要文字描述动作**
- 布局：左侧当前意图（32×32 大图标+数字）→ 金色箭头 → 右侧分支框
- 分支框内每行：概率% + 约束 ≤N + 意图图标(24×24) + 数字
- 底部回环箭头 ↩ 表示循环

```
┌─── Jaw Worm ──────────────────────┐
│                                    │
│  [攻击图标]  ─────→  ┌──────────┐ │
│     11               │ 25% ≤1   │ │
│                      │ [攻击] 11│ │
│                      │ 45% ≤1   │ │
│                      │ [攻防]   │ │
│                      │ 30% ≤2   │ │
│                      │ [防御] 7 │ │
│                      └──────────┘ │
│                           ↩       │
└────────────────────────────────────┘
```

- 面板 bg rgba(0.15,0.18,0.25,0.9)、border 1px rgba(0.5,0.5,0.6,0.6)、radius 6px
- 当前状态框：bg rgba(0.15,0.2,0.3,0.8) + aqua #2AEBBE 左边框 3px
- 分支框：bg rgba(0.1,0.12,0.18,0.8)、border 1px rgba(0.4,0.5,0.7,0.5)
- 概率值 gold #EFC851 12px、约束标签 cream 11px、箭头/回环 gold #EFC851

### 3.11 个人生涯统计

集成到游戏现有的 **百科大全 → 角色数据 → 统计页面**，位于**"总体数据"面板下方**（非独立面板）。

**统计指标**（按角色分组，含总体）：

| 类别 | 指标 |
|------|------|
| **胜率趋势** | 最近 10 / 50 / 全部 局的滚动胜率 |
| **死因排行** | 最常死于的遭遇 Top 5（次数 + 百分比），按 Act 分组 |
| **路径统计**（每 Act 平均） | 获取卡牌数量、购买卡牌数量、删除卡牌数量、升级卡牌数量 |
| **路径统计**（每 Act 平均） | ? 房间数、小怪战斗数、精英战斗数、商店数 |
| **先古遗物选择率** | 下拉选择先古之民 → 显示每个选项的选择率/次数 + 每个遗物的选率/次/胜率/浮动 |
| **Boss 战损** | 下拉选择 Boss → 显示平均战损 HP + 死亡率 |

**数据说明**：所有显示数据为**用户个人生涯数据**。

**交互**：鼠标悬停任一统计项时，tooltip 显示**社区对比数据**（社区数据受 F9 筛选面板条件影响）。

### 3.16 商店价格显示

1. 地图/房间界面中，鼠标悬停在**金币图标**上时弹出商店价格表面板
2. 显示内容：
   - 当前删牌费用（递增：75 + 25 × 已删次数）
   - 各稀有度卡牌价格范围（普通/非普通/稀有）
   - 各稀有度遗物价格范围（普通/非普通/稀有/商店限定）
   - 各稀有度药水价格范围
   - 打折说明（50% off）、无色牌加价说明（+15%）
3. 数据来源：游戏内 `MerchantCardEntry.GetCost()`、`RelicModel.MerchantCost`、`MerchantPotionEntry.GetCost()`、`MerchantCardRemovalEntry.CalcCost()`
4. 价格范围计算：基础价格 × ±随机波动（卡牌/药水 ±5%，遗物 ±15%）

- InfoMod 风格深色面板
- 标题 white 14px + 删牌费用 red + 副标题 gray 11px
- 表格布局：类别名（gold/aqua）× 稀有度列 → 价格范围 cream 12px

```
┌───────────────────────────────────┐
│ Shop Prices       删牌费用: 75   │
│ 打折50%, 无色牌贵15%             │
│ ─────────────────────────────── │
```

## 4. PRD-00 补全项

### 4.1 多乘法修正器拆分约束

**问题**：多个独立乘区修正器同时存在时，各自贡献公式 `finalDmg - finalDmg/multiplier` 之和可能超过实际 bonus。

**示例**：打击基础伤害 6，易伤 ×1.5，双倍伤害 ×2。
- 总伤害 = 6 × 1.5 × 2 = 18，实际 bonus = 18 - 6 = 12
- 易伤贡献 = 18 - 18/1.5 = 18 - 12 = 6
- 双倍伤害贡献 = 18 - 18/2 = 18 - 9 = 9
- 各自贡献之和 = 6 + 9 = 15 > 实际 bonus 12

**解决方案**：强制约束 `DirectDamage + ModifierDamage = TotalDamage`。当修正器贡献之和超过 `TotalDamage - baseDamage` 时，按比例缩放各修正器贡献：
```
缩放系数 = (TotalDamage - baseDamage) / 修正器贡献之和
每个修正器实际贡献 = 原贡献 × 缩放系数
```

### 4.2 Confused/SneckoEye 随机改费

蛇眼 SneckoEye 的随机改费效果 Confused **计入**能量贡献计算。按照实际卡牌使用费用与原版卡牌费用的差值计算，因此可能出现**能量贡献为负**的情况。

涉及来源实体：
- 先古遗物：异蛇之眼 Snecko Eye
- 假商人事件遗物：异蛇之眼??? Snecko Eye???
- 药水：异蛇之油 Snecko Oil

### 4.3 Enlightenment 部分降费

开悟 Enlightenment 降低手牌费用到 1 点。**部分降费也追踪**。例如一张 3 费牌被降到 1 费，节省 2 能量归属到 Enlightenment。

### 4.4 Buffer 防御计算示例

PRD-00 §4.5.1 防御计算第 5 步 Buffer 的公式补充示例：

> 敌人造成 5 点伤害 ×4 次，我方有 2 层缓冲 Buffer 和 7 点格挡。
> - 第 1 段 5 点伤害，格挡吸收 5 点（格挡剩余 2）
> - 第 2 段 5 点伤害，格挡吸收 2 点，剩余 3 点穿透 → Buffer 层 1 抵消 3 点
> - 第 3 段 5 点伤害 → Buffer 层 2 抵消 5 点
> - 第 4 段 5 点伤害 → 无 Buffer，玩家受 5 点伤害
> - Buffer 防御贡献 = 3 + 5 = 8 点
