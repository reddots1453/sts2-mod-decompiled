# PRD-02: 战斗统计修复与测试

> 版本：2.0 | 日期：2026-04-07  
> 依赖：PRD-00 §4.5.1（战斗贡献计算需求）  
> 涉及模块：CombatTracker、ContributionMap、CombatHistoryPatch、ContributionChart

---

## 1. 概述

战斗贡献统计是本 Mod 最复杂的系统（~2500 行补丁代码、60+ 力量钩子、25+ 遗物钩子）。经人工测试发现多处数据遗漏和不准确。本 PRD 定义系统性测试矩阵和修复标准，确保所有已知场景下统计数据准确。

**原则**：不新增统计维度，只修复和补全现有统计项的准确性。

## 2. 术语表

> 完整术语定义见 PRD-00 §2，此处仅列出本 PRD 中高频使用的术语。

| 术语 | 定义 |
|------|------|
| **贡献类别** | 伤害（Damage）、防御（Defense）、抽牌（Draw）、能量（Energy）、辉星（Stars）、治疗（Healing）六大类 |
| **贡献来源类型** | 卡牌（Card）、遗物（Relic）、药水（Potion）三种 |
| **直接伤害 DirectDamage** | 卡牌效果直接造成的伤害 |
| **归因伤害 AttributedDamage** | 由间接效果（中毒、荆棘、充能球等）造成的伤害，溯源到施加来源 |
| **修正伤害 ModifierDamage** | 由数值修正器（力量、易伤等）贡献的伤害增量 |
| **升级增量 UpgradeDamage/UpgradeBlock** | 战斗内临时升级带来的额外伤害/格挡，归因到执行升级的来源 |
| **加法修正器** | 以固定值叠加的修正器（如力量、灵巧），贡献 = 增量值 |
| **乘法修正器（独立乘区）** | 拥有独立乘算区间的修正器（如双倍伤害），贡献 = 总伤害 - 总伤害/(1+系数) |
| **乘法修正器（非独立乘区）** | 共享乘算区间的修正器（如残酷 Cruelty 改变易伤基础系数），贡献按区间内系数占比分配 |
| **FIFO 格挡池** | 格挡归因的先进先出队列：先获得的格挡先被消耗 |
| **FIFO 层数归因** | 增伤/减伤状态的多来源层数，先施加的先在当前回合生效 |
| **LIFO Osty 栈** | Osty HP 归因的后进先出栈：最后恢复的 HP 先被伤害消耗 |
| **上下文栈** | ContributionMap 中追踪"谁在生效"的栈结构，Prefix 入栈、Postfix 出栈 |
| **来源归因优先级** | 明确卡牌 → 当前打出卡牌 → 当前药水 → 当前遗物钩子 → 当前能力牌来源 → 当前充能球引导来源 → 未追踪 |
| **末杀修正** | 最后一击的特殊处理：`IsEnding` 标记导致常规补丁跳过，需 KillingBlowPatcher 补偿 |
| **子来源 Sub-bar** | 由某个来源生成/变形的卡牌的贡献，显示为主条目下的缩进子条目 |
| **未追踪** | 无法归因到具体来源的效果，单独归类并写入日志 |

## 3. 测试方法

### 3.1 测试执行方式

由于 Harmony 补丁依赖游戏运行时，无法做传统单元测试。测试通过以下方式进行：

```
设计测试场景（使用 DevTools Mod 构造牌组/遗物/敌人）
  → 进入战斗，执行预定操作
  → 战斗结束后查看贡献面板
  → 手动比对预期值与实际值
  → 不匹配的场景记录为 Bug → 修复 → 回归测试
```

### 3.2 测试场景构造

使用 `sts2_dev_tools` Mod（F7 解锁全部）+ 游戏内自定义模式构造：
- 选择特定角色
- 通过自定义牌组控制手牌
- 选择特定遭遇控制敌人
- 装备特定遗物/药水

### 3.3 验证数据读取

贡献面板在战斗结束后自动弹出，显示：
- **本场战斗** Tab：本次战斗的贡献明细
- **本局汇总** Tab：累计所有战斗的贡献

验证时需检查：
- 每个来源的贡献数值是否正确
- 子条目（Sub-bar）是否正确归因到父来源
- 各类别总和是否与战斗中实际发生量一致（不多不少）
- 修正器增量是否从总量中正确拆分
- 遗漏的来源是否标记为"未追踪"并出现在日志中

## 4. 测试矩阵

### 4.1 伤害归因 — 直接伤害

| # | 场景 | 操作 | 预期 |
|---|------|------|------|
| D1 | 单体攻击卡 | 打出"打击"对敌人造成伤害 | 打击的 DirectDamage = 实际伤害值 |
| D2 | AOE 攻击卡 | 打出旋风斩对所有敌人造成伤害 | 旋风斩的 DirectDamage = 所有敌人受到伤害之和 |
| D3 | 多段攻击卡 | 打出"连击"命中 3 次 | 连击的 DirectDamage = 3 次伤害之和 |
| D4 | 末杀伤害 | 用一张卡打出 20 伤害，敌人只剩 5 HP | 该卡的 DirectDamage += 5（实际造成），而非 20（面板值） |
| D5 | 末杀不重复 | 同 D4，检查该伤害只被计算一次 | DamageResult 通过 identity hash 去重，不被 DamageReceived 和 KillingBlowPatcher 双重计入 |
| D6 | 遗物被动伤害 | 水银沙漏每回合对所有敌人造成 3 伤害 | 水银沙漏 DirectDamage += 3 × 敌人数 |

### 4.2 伤害归因 — 数值修正拆分

> 对应 PRD-00 §4.5.1 伤害归因需求中的数值修正拆分公式。

| # | 场景 | 操作 | 预期 |
|---|------|------|------|
| M1 | 加法修正：力量 | 有 3 力量（来源：力量药水）时打出"打击"（基础 6） | 打击 DirectDamage = 6，力量药水 ModifierDamage = 3 |
| M2 | 加法修正：灵巧 | 有 2 灵巧（来源：能力牌 A）时打出"防御"（基础 5） | 防御 EffectiveBlock 基于 7 格挡，能力牌 A ModifierBlock = 2 对应的有效格挡量 |
| M3 | 独立乘区：易伤 | 敌人有易伤（系数 0.5，来源：痛击 Bash），打出"打击" | 痛击 ModifierDamage = 总伤害 - 总伤害/(1+0.5) |
| M4 | 非独立乘区：残酷 | 敌人有易伤（基础系数 0.5）+ 残酷 Cruelty（系数 0.25），打出"打击" | 痛击的贡献 = 区间总贡献 × 0.5/(0.5+0.25)，残酷的贡献 = 区间总贡献 × 0.25/(0.5+0.25) |
| M5 | 多来源增伤层数 FIFO | 先由卡牌 A 施加 2 层易伤，再由卡牌 B 施加 1 层易伤 | 当前回合：卡牌 A 的层数先生效（FIFO），贡献优先归因到 A |
| M6 | 升级增量 | 通过"武装"升级"打击"（6→9），打出升级后的打击 | 打击 DirectDamage = 6，武装 UpgradeDamage = 3 |
| M7 | 升级增量 + 力量叠加 | 有 3 力量 + 打击被升级（6→9），打出一次 | 打击 DirectDamage = 6，武装 UpgradeDamage = 3，力量来源 ModifierDamage = 3 |
| M8 | 多个独立乘区叠加 | 同时有易伤（×1.5）+ 双倍伤害（×2），打出"打击" | 各乘区独立计算贡献，总 ModifierDamage 不重复 |

### 4.4 伤害归因 — 来源优先级

> 对应 PRD-00 §4.5.1 来源归因优先级。

| # | 场景 | 操作 | 预期 |
|---|------|------|------|
| P1 | 明确卡牌来源 | 打出攻击卡，触发荆棘等连锁效果 | 连锁效果中若有明确 cardSource 字段，按该字段归因 |
| P2 | 药水上下文 | 使用火焰药水直接造成伤害 | 归因到火焰药水（当前使用的药水） |
| P3 | 遗物钩子上下文 | 遗物被动触发伤害（如水银沙漏） | 归因到该遗物（当前执行钩子的遗物） |
| P4 | 能力牌间接效果 | 荆棘能力牌的反伤触发 | 归因到施加荆棘效果的那张能力牌 |
| P5 | 无上下文 | 无法追溯来源的间接效果 | 标记为"未追踪"，写入运行日志 |

### 4.5 防御归因 — 计算流水线

> 对应 PRD-00 §4.5.1 防御归因需求中的 5 步计算顺序。  
> 敌人预计初始伤害 D0 → 逐步减免 → 剩余实际受伤。

#### 第 1 步：力量削减减伤

| # | 场景 | 操作 | 预期 |
|---|------|------|------|
| DEF-1a | 单次力量削减 | 降低敌人 2 力量后，敌人攻击（单段） | 削减来源 MitigatedByStrReduction = min(力量削减量, 敌人单段原伤害) × 攻击段数 |
| DEF-1b | 多来源力量削减 | 卡牌 A 削减 2 力量 + 卡牌 B 削减 1 力量，敌人攻击 | 按各自削减量比例分配减伤（A 得 2/3，B 得 1/3） |

#### 第 2 步：乘法修正器减伤（虚弱等）

| # | 场景 | 操作 | 预期 |
|---|------|------|------|
| DEF-2a | 虚弱减伤 | 敌人有虚弱（系数 α，来源：某卡牌），攻击 D1 | 虚弱来源 MitigatedByDebuff = D1 × α |
| DEF-2b | 非独立减伤修正器 | 纸鹤 Paper Krane 改变虚弱系数（虚弱 β₀ + 纸鹤 β₁），敌人攻击 | 虚弱贡献 = 区间总减免 × β₀/(β₀+β₁)，纸鹤贡献 = 区间总减免 × β₁/(β₀+β₁) |
| DEF-2c | 多来源虚弱层数 FIFO | 先由卡牌 A 施加 2 层虚弱，再由卡牌 B 施加 1 层 | 卡牌 A 的层数先生效（FIFO），贡献优先归因到 A |
| DEF-2d | 独立乘区减伤（Colossus） | 巨像 Colossus：玩家 buff（由巨像卡施加），当有易伤的敌人攻击玩家时伤害 ×0.5。敌人有易伤状态并攻击玩家 | Colossus 来源 MitigatedByBuff += actualDamage（0.5x 乘数，prevented = actualDamage） |

#### 第 3 步：无实体增益减伤

| # | 场景 | 操作 | 预期 |
|---|------|------|------|
| DEF-3a | 无实体减伤 | 有无实体（来源：某卡牌），敌人攻击 D2（单段） | 无实体来源 MitigatedByBuff = D2 - min(1, D2)；多段时 = D2 - min(1, D2) × 段数 |

#### 第 4 步：格挡减伤

| # | 场景 | 操作 | 预期 |
|---|------|------|------|
| DEF-4a | 基础格挡 | 打出"防御"获得 5 格挡，敌人攻击 D3=3 | 防御 EffectiveBlock = 3（被消耗的部分） |
| DEF-4b | 格挡溢出 | 打出"防御"获得 5 格挡，敌人攻击 D3=10 | 防御 EffectiveBlock = 5（全部消耗） |
| DEF-4c | FIFO 顺序 | 先打"防御 A"5 格挡 + 再打"防御 B"5 格挡，敌人攻击 D3=7 | A 的 EffectiveBlock = 5，B 的 EffectiveBlock = 2 |
| DEF-4d | 格挡修正拆分 | 有 2 灵巧（来源：能力牌 X）时打出"防御"（基础 5），敌人攻击 7 | 灵巧提供的 2 格挡中实际吸收的部分归属能力牌 X（ModifierBlock），其余归属防御 |
| DEF-4e | 遗物被动格挡 | 蛇眼 Orichalcum 回合结束获得 6 格挡（无格挡时），敌人攻击 4 | 蛇眼 EffectiveBlock = 4 |

#### 第 5 步：Buffer 增益减伤

| # | 场景 | 操作 | 预期 |
|---|------|------|------|
| DEF-5a | Buffer 减伤 | 有 Buffer 1 层（来源：某卡牌），格挡为 0，敌人单段攻击 20 | Buffer 来源 MitigatedByBuff = 20（整段被 Buffer 抵消） |
| DEF-5b | Buffer 多段 | 有 Buffer 2 层，敌人 3 段攻击（每段 10），格挡已在第 4 步消耗 | Buffer 抵消前 2 段，剩余 1 段穿透；Buffer 来源 MitigatedByBuff = 20 |
| DEF-5c | 自伤 | 打出放血 Bloodletting（自伤 3 HP） | 放血 SelfDamage = 3，显示为红色负防御，不参与防御百分比占比计算 |

#### 防御总量一致性

| # | 场景 | 预期 |
|---|------|------|
| DEF-V1 | 任意战斗 | D0（敌人预计伤害总和）- 实际受伤 = 力量削减 + 乘法减伤 + 无实体 + 格挡 + Buffer |
| DEF-V2 | 无任何防御手段 | 防御类别总值 = 0，实际受伤 = D0 |

### 4.6 Osty 特殊场景（亡灵契约师）

### 4.10 容错与一致性

| # | 场景 | 预期 |
|---|------|------|
| F1 | 未追踪来源 | 无法归因的效果记入"未追踪"类别，不丢弃数据；运行日志输出具体未追踪信息 |
| F2 | 未处理的力量/遗物 | 不崩溃，graceful 降级，贡献归入未追踪 |

| F4 | 伤害总量一致 | 伤害图表总和 = DirectDamage + AttributedDamage + ModifierDamage + UpgradeDamage = 战斗中敌人实际受到的总伤害 |
| F5 | 防御总量一致 | 防御图表总和（不含 SelfDamage）= 敌人意图伤害总和 - 玩家实际受伤 |
| F6 | 上传精简一致 | 上传到服务器的数据 = 本地面板显示的聚合值（合并后的总伤害、总格挡等） |

## 5. 验收标准

### AC-01: 直接伤害准确
- [ ] D1-D6 全部场景通过手动验证
- [ ] 末杀伤害不遗漏、不重复计算

- [ ] 多来源层数按 FIFO 归因

### AC-03: 间接伤害准确
- [ ] I1-I5 全部场景通过手动验证
- [ ] 中毒、荆棘、充能球、Doom 均正确溯源到施加来源

- [ ] P1-P5 场景通过验证
- [ ] 无法归因时标记为"未追踪"而非丢弃

## 7. 技术约束

- 测试只能通过游戏内手动操作进行（Harmony 补丁不支持脱离游戏的单元测试）
- 可借助 DevTools Mod 构造测试场景（解锁全部卡牌/遗物）
- 修改仅限 `src/Collection/` 和 `src/Patches/` 目录
- 不新增贡献类别，不修改 ContributionAccum 的字段结构（除非 PRD-00 明确要求）
- 每次修复后需回归验证之前通过的场景
