# STS2 Community Stats - 社区数据共享 Mod

**版本**: 1.0.0  
**适用**: Slay the Spire 2 (Early Access v0.99.x+)  
**类型**: 纯信息展示，不影响游戏平衡 (`affects_gameplay: false`)

---

## 这个 Mod 做什么？

Community Stats 收集所有订阅玩家的游玩数据，汇总后在游戏界面中实时展示社区统计信息。帮你做出更明智的选择 —— 或者，看看你的选择有多离谱。

---

## 功能一览

### 1. 选卡界面 — 社区选取率 & 胜率

在卡牌奖励界面，每张卡牌下方显示：

```
Pick 72% | Win 54%
```

- **Pick**: 社区中该卡出现在奖励中时被选取的比例
- **Win**: 选取该卡的玩家最终通关的比例
- 颜色编码：绿色(高胜率) / 黄色(中等) / 红色(低胜率)

### 2. 事件选项 — 社区选择率 & 胜率

在事件选项按钮旁显示：

```
Chosen 64% | Win 48%
```

- 每个选项有多少比例的玩家选择了它
- 选择该选项后的通关率

### 3. 地图节点 — 遭遇战危险度

在战斗类型的地图节点上叠加显示：

```
Death 3.2% | Avg DMG 18
```

- **Death**: 社区中在该遭遇战死亡的比例
- **Avg DMG**: 平均受到的伤害

覆盖 Monster / Elite / Boss 三种战斗节点。

### 4. 战斗贡献统计图

**每场战斗结束后自动弹出**（可关闭），展示本场战斗中每张卡牌和每个遗物的贡献：

```
本场战斗贡献 (vs Gremlin Nob)
────────────────────────────
伤害
  Strike     ████████████████  64  (42%)
  Bash       ██████████  40  (26%)
  [R] Vajra  █████  20  (13%)      ← 遗物贡献
  Poison     ████  16  (10%)       ← 间接伤害归因
────────────────────────────
格挡
  Defend     ████████████  30  (60%)
  [R] Orich  ██████  15  (30%)
  Shrug      ██  5  (10%)
────────────────────────────
抽牌: 3 | 能量: 1 | 治疗: 5
```

**特色**：
- 卡牌(蓝色) 和遗物(金色) 在同一图表中混合排序
- 间接伤害追踪：中毒、力量等 buff 造成的伤害追溯到施加该效果的卡牌/遗物
- `[R]` 前缀标识遗物贡献

### 5. 全局贡献汇总

按 **F8** 可随时打开贡献面板，包含两个标签页：

| 标签 | 内容 |
|------|------|
| **上场战斗** | 最近一场战斗的详细贡献 |
| **本局汇总** | 本次 Run 所有战斗的累计贡献 |

### 6. 筛选面板

按 **F9** 打开筛选面板，自定义你想看到的社区数据范围：

| 筛选项 | 说明 |
|--------|------|
| **自动匹配进阶** | 自动匹配你当前的进阶等级附近的数据 |
| **最低进阶** | 手动设置进阶范围下限 (0-20) |
| **最高进阶** | 手动设置进阶范围上限 (0-20) |
| **游戏版本** | 当前版本 / 全部版本 |

修改筛选条件后点击 Apply，数据会在后台重新加载。

### 7. 商店购买记录

记录你在商店中的购买行为（卡牌、遗物、药水），上传后用于计算社区的商店购买率。

### 8. 卡牌移除 & 升级追踪

- 通过商店或事件**移除**卡牌时自动记录
- 通过火堆或事件**升级**卡牌时自动记录
- 这些数据用于计算社区的卡牌移除率和升级率

## 数据说明

### 什么时候上传数据？

**仅在每次 Run 结束时上传一次**（无论胜利或失败）。不会在游戏过程中持续上传。

### 上传了哪些数据？

| 数据 | 说明 |
|------|------|
| 角色 & 进阶等级 | 用于分类统计 |
| 胜败结果 & 到达层数 | 用于计算胜率 |
| 每次卡牌奖励的选择 | 出现了哪些卡、选了哪张（或跳过）|
| 事件选项选择 | 选了哪个选项 |
| 商店购买记录 | 买了什么、花了多少金币 |
| 卡牌移除/升级记录 | 移除/升级了哪张卡 |
| 每场遭遇战数据 | 敌人ID、受到伤害、回合数、是否死亡 |
| 战斗贡献数据 | 每张卡/遗物的伤害/格挡/抽牌等贡献 |
| 最终卡组 & 遗物 | Run 结束时的卡组和遗物列表 |
| 游戏版本 & Mod版本 | 用于版本兼容性管理 |

### 不会上传的数据

- **不会上传** Steam ID 或任何个人身份信息
- **不会上传** 你的系统信息或其他游戏数据
- **不会上传** 任何截图或录像

### 数据什么时候加载？

- **Run 开始时**：后台批量预加载当前角色的全部统计数据（约 30KB 压缩后）
- **游戏过程中**：从本地缓存读取，0 延迟
- **缓存未命中时**：自动回退到网络请求，显示 "Loading..." 后更新

### 离线模式

如果无法连接服务器：
- 上传数据自动保存到本地队列，下次联网时重传
- 上次成功加载的统计数据保留在磁盘缓存中（24小时有效），离线也能查看

---

## 数据存储位置

本地缓存和设置存储在：

```
%AppData%/sts2_community_stats/
├── cache/          # 磁盘缓存（统计数据）
├── pending/        # 待上传队列（离线时）
└── settings.json   # 筛选设置
```

## 安装

1. 将 `sts2_community_stats` 文件夹放入游戏目录的 `mods/` 文件夹下
2. 启动游戏，Mod 自动加载
3. 开始一局新 Run，统计数据会自动加载并显示

### 文件结构

```
mods/sts2_community_stats/
├── manifest.json
└── bin/
    └── sts2_community_stats.dll
```

## 常见问题

**Q: 这个 Mod 会影响游戏平衡吗？**  
A: 不会。Mod 标记为 `affects_gameplay: false`，仅显示信息，不修改任何游戏逻辑。

**Q: 数据显示 "No data" 是怎么回事？**  
A: 可能是网络连接问题，或该卡牌/事件的样本量还不够。随着更多玩家使用，数据会逐渐丰富。

**Q: 我可以关闭战斗结束后的自动弹窗吗？**  
A: 弹窗出现后点击其他区域即可关闭。也可以用 F8 随时手动控制开关。

**Q: 筛选条件改变后数据需要多久刷新？**  
A: 后台立即重新加载，通常 1-3 秒内完成。期间如果打开界面，会自动按需请求。

**Q: 支持多人模式吗？**  
A: 支持。Run 数据中会包含玩家数量信息，你可以通过筛选条件过滤多人/单人数据。

## 版本兼容

Mod 自动检测当前游戏版本。Early Access 期间如果卡牌/遗物被重命名，服务端会自动进行 ID 迁移映射，确保跨版本数据统计的一致性。
