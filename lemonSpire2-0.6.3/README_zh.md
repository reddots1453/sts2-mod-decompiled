# Lemon Spire 2 / 柠檬塔2

受薄荷尖塔启发的模组，主要面向杀戮尖塔2（尤其是多人模式）提供一些便利功能。

由于我使用了 Alchyr 的 [BaseLib](https://github.com/Alchyr/BaseLib-StS2/releases) 来解决 Linux 兼容性问题，因此似乎不安装
BaseLib 就无法正常使用这个模组了。

## 功能

### 颜色 / 绘图相关

修改您的颜色！

同时影响：鼠标的颜色、地图画画的颜色、左侧信息栏的名字颜色，聊天信息的名字颜色。

(TODO)似乎有点太扎眼/破坏辨识度了，可能得找个办法修改内部颜色而不修改外侧描边的颜色。

### 简易队友连携显示

在界面上直观显示队友是否抽到了“多人专属牌”，或有牌能造成协助团队的 Buff 效果。

- 常规 Buff 会直接显示为对应的 Buff 图标。目前仅支持了 易伤、脆弱、勒脖 三种。
- 多人专属牌会统一显示为“握手”图标。
- 计划未来让扩展性更好些。目前特化的图标还不能直接覆盖握手图标，如果想给某张特定的牌（比如“夹击”）做特化图标就需要更改握手图标的逻辑，这实在太不方便了。

点击自己的小图标可以让它全队虚化来表示“我打不出来”以提示队友。

阅读 `lemonSpire2.SynergyIndicator.Models.IndicatorProvider`。

### 成套多人聊天

按 Tab 键可展开或折叠聊天框，支持鼠标拖拽。

安全的 BBCode ，可以通过 [b] [color=red] 等标签来修改文本样式。未闭合的标签会被自动尾闭合。

底层做了一个还算行的 Segment 消息段系统，自定义样式似乎是可行的（虽然目前还没有做）。

欢迎参考 `lemonSpire2.Chat` 来了解如何添加新的消息段类型，或者修改现有的消息段样式。

另外也可以尝试注册新的 Intent 到 ChatStore 来实现更多功能。或者麻烦帮我把 TextSubmit 的 Intent 也拆碎一下。

#### 发送万物

使用 Alt + 左键点击 可以发送遗物、牌、buff 的信息到多人聊天中，生成悬停提示框供队友查看。

这一功能的 Vibe Codebase 感谢 [sts2_typing by Shiroim](https://github.com/Shiroim/sts2_typing), MIT License

Alt + 右键点击 可以发送 “当前的 HoverTip”，适合用来分享事件中即将塞给你的牌和遗物，或者手牌上附加的 Buff 等信息。

- 右键功能在读取卡牌时很准确，但是游戏底层关于事件和卡牌文本的存储略微有点复杂，所以我也在偷懒。
- 目前的实现方式非常粗暴——直接去读 HoverTip 里的字。这种做法导致后续无法提取出具体的对象信息，可扩展性极差，还会破坏 i18n。
- 另外，这玩意目前只会读取 1 个 HoverTip，这我得思考一下怎么做更好。

### 简易贡献统计

多人模式下，当您将鼠标悬停在左侧角色状态上时，使用 HoverTip 显示玩家的伤害贡献、buff 贡献、“额外伤害”贡献（即易伤、脆弱等提升的伤害，追溯到
buff 源有点困难所以就统计了这个）。

- 其实追溯 Buff 源头本身并不难，问题在于相同 Buff 叠加时游戏逻辑并不修改“buff 源”属性，这导致后续叠加 Buff
  的玩家吃不到贡献算分，很不公平。所以我摆烂了。
- 总不能让我做另一套 History 系统吧。

这里的 HoverTip 机制同样对扩展开放。想了解如何添加新的悬停统计信息，请查阅 `NPlayerState.ITooltipProvider`。

已有的贡献统计代码位于 `StatsTracker` 命名空间下，欢迎参考。

### 战斗内状态悬浮面板

在原版游戏中，点击左侧的角色状态会直接弹出一个全屏面板，展示完整的牌组和遗物列表——这对单次战斗的直接帮助其实比较小，而且开关这个面板挺麻烦的。

因此，我们将“点击角色状态”的功能改为了呼出一个小型的悬浮面板，直接展示队友本回合的手牌和现有药水。
（原本的全屏面板现在改成了通过**双击**或**右键**打开）。

还能显示队友面对的卡牌奖励。这包括战后的奖励或者星系仪等遗物给予的奖励，甚至包括特殊奖励（灯火钥匙 和 拿回被偷的牌）

在商店，悬浮面板会显示队友的金币和他面对的商店物品。 配合别的赠送卡牌和物品的 Mod 使用才有实际意义，我们只提供了一个显示面板。

该功能同样支持扩展，请查阅 `lemonSpire2.NPlayerState.Panel` 来了解如何往面板里塞入新的信息。

顺带一提，这个悬浮面板目前有的这些东西也是支持 Alt + 点击 发送的。
